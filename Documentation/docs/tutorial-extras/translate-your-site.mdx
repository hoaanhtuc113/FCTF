# Localization Policy

Current documentation policy is English-first (`defaultLocale: en`).

This keeps terminology consistent across Admin, Architecture, Install, and Contestant guides.

## Current state

- Active locale: `en`
- Source of truth: `docs/`
- No translated docs tree is currently published

## If multilingual docs are introduced later

1. Add locale entries in `docusaurus.config.ts`.
2. Establish translation review ownership per locale.
3. Keep approved baseline pages synchronized first:
   - `docs/product-and-features/admin/*`
   - `docs/architecture/*`
   - `docs/install-and-ops/*`
4. Add localization QA to release checklist (links, images, and terminology parity).

## Non-goal

Do not mix languages inside the same operational page.

If translation is unavailable for a page, keep that page in English until the full translation is reviewed.
