# Manage Documentation Versions

FCTF currently publishes a single active documentation line for v4.

If formal multi-version docs are needed, follow this workflow.

## Versioning trigger

Create a new docs version when:

- A major platform release changes operational behavior
- Backward compatibility needs separate runbooks

## Create a version snapshot

```bash
npm run docusaurus docs:version <version-tag>
```

Example:

```bash
npm run docusaurus docs:version 4.1.0
```

## Versioning rules

1. Freeze the old version after release.
2. Continue active changes in `docs/` (current line).
3. Update navbar labels so operators can identify the active line.
4. Keep architecture and install pages synchronized with release tags.

## Verification

After creating a version:

1. Build docs locally.
2. Validate version selector and links.
3. Verify old version pages still render screenshots correctly.
