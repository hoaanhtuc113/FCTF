# Team Collaboration and Support

This section explains participant collaboration and communication features during competition.

## Team Profile and Progress Context

**Participant goal:** Keep team state and role context accurate.

**Operational steps**

1. Review team profile, member list, and role ownership.
2. Track rank and score progression over time.
3. Update personal profile and credentials when needed.
4. Coordinate challenge assignment across members.
5. Reconfirm context before critical submissions.

**Expected result:** Team coordination is clear and low-friction.

## Scoreboard Access and Interpretation

**Participant goal:** Use scoreboard as a strategic decision tool.

**Operational steps**

1. Open scoreboard and review full ranking table.
2. Read score, solved count, and rank movement indicators.
3. Apply bracket filters for peer-group comparison.
4. Track rank changes after solve milestones.
5. Interpret freeze periods correctly when rank updates pause publicly.
6. Adjust challenge priorities based on standings gap.

**Expected result:** Ranking visibility directly supports competition strategy.

## Contestant Ticket Support

**Participant goal:** Escalate blockers with clear evidence and fast resolution.

**Operational steps**

1. Create ticket with concise issue summary.
2. Attach context (challenge, time, screenshots, observable errors).
3. Monitor ticket status and respond to follow-up requests.
4. Validate proposed fix in your workflow.
5. Close ticket after practical confirmation.

**Expected result:** Support response cycle is faster and clearer.
