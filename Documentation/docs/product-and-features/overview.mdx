# Product & Features Overview

FCTF combines challenge authoring, secure runtime orchestration, contestant solve workflows, and operational governance in one platform.

This page is the entry map for feature documentation.

:::tip[How to use this section]
Read this page first, then open the role-specific pages in the sidebar for operational details.
:::

## Role Tracks

| Track | Primary audience | Canonical pages |
| --- | --- | --- |
| Admin Features | Admin, Jury, Challenge Writer | `docs/product-and-features/admin/*` |
| Contestant Features | Contestants and team captains | `docs/product-and-features/contestant/*` |

The Admin track and Architecture/Install tracks are the approved baseline for platform operations. Other pages should stay aligned with those baseline semantics.

## Product Goals

- Deliver a reliable contestant solving experience under high concurrency.
- Give operations teams full control over challenge lifecycle and runtime environments.
- Preserve fairness with server-side enforcement, traceability, and policy controls.
- Support post-event analysis through analytics, logs, and audit evidence.

## Feature Families

### Admin Families

- Accounts, identity governance, and compliance controls
- Challenge authoring, deployment, and version lifecycle
- Live runtime control, monitoring, and audit-grade log surfaces
- Scoring analytics, reward queries, and event governance

### Contestant Families

- Access and challenge discovery
- Solve workflow (start, access token, submit)
- Team collaboration (profile, scoreboard, activity logs)
- Support operations (ticket create/list/view/delete)
- Fair-play controls and security guardrails

## Documentation Quality Standard

Keep each feature page consistent with approved tracks by using this structure:

1. Scope and purpose
2. UI and operation flow
3. Key controls and constraints
4. Expected outcomes
5. Evidence sources (logs, status, API behavior)
