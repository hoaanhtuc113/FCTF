# Build and Validate Documentation

Use this sequence before publishing documentation changes.

## 1) Install dependencies

```bash
npm install
```

## 2) Run local development server

```bash
npm run start
```

Verify that changed pages load correctly and screenshot paths resolve.

## 3) Run production build

```bash
npm run build
```

This step catches:

- Broken links
- Invalid MDX syntax
- Front matter errors

## 4) Preview production output

```bash
npm run serve
```

Check navigation, search, and page rendering in the production build.

## 5) Merge policy

Documentation and behavior changes should be merged together in the same pull request.
