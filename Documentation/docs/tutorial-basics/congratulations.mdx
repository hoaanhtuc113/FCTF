# Documentation Completion Checklist

Use this checklist before closing a documentation task.

## Content quality

1. Page is in English and uses platform terminology consistently.
2. Workflow steps match implemented UI and API behavior.
3. Constraints and policy controls are clearly documented.

## Technical quality

1. Front matter is valid.
2. Internal links resolve.
3. Screenshot paths exist and render.
4. `npm run build` passes.

## Consistency quality

1. Content aligns with approved baseline sections:
	- `docs/product-and-features/admin/*`
	- `docs/architecture/*`
	- `docs/install-and-ops/*`
2. No outdated Docusaurus template text remains in operational pages.
