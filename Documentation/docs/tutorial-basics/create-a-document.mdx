# Create a Documentation Page

FCTF docs are manually curated in `sidebars.ts`.

## Standard file template

```mdx title="docs/example-page.mdx"
---
title: Example Page
description: One-line summary used by search and SEO.
sidebar_position: 1
---

# Example Page

Write operationally accurate content based on implemented behavior.
```

## Authoring rules

1. Use English for all primary documentation pages.
2. Prefer explicit operational steps over abstract language.
3. Add screenshots only when they clarify a workflow.
4. Keep links relative to the docs tree.
5. Update `sidebars.ts` when adding navigable docs.

## Required validation

Before merging documentation changes:

1. Run local docs build.
2. Verify no broken links.
3. Verify front matter fields are valid.
4. Ensure terminology matches approved tracks:
   - `docs/product-and-features/admin/*`
   - `docs/architecture/*`
   - `docs/install-and-ops/*`
