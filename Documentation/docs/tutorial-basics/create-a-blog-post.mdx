# Publish Release Notes (No Blog Mode)

The Docusaurus blog is disabled in this repository (`blog: false` in `docusaurus.config.ts`).

For release communication, use documentation pages instead of blog posts.

## Recommended release-note locations

- Update `docs/intro.mdx` with current release metadata.
- Update affected feature pages directly with change notes.
- If needed, add a dedicated release note page under `docs/` and link it from `sidebars.ts`.

## Why this policy exists

- Keeps technical truth close to operational pages.
- Avoids stale dual sources (blog vs docs).
- Preserves a single review path for platform changes.
