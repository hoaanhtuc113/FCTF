# Create a Standalone Page

Use `src/pages` only for site-level pages that are not part of the docs sidebar.

Examples:

- `src/pages/index.tsx`: website homepage
- `src/pages/xyz.mdx`: standalone informational page

## When to use `src/pages`

Use this path for:

- Landing pages
- Legal or policy pages
- Compatibility or redirect pages

Do not use this path for feature documentation. Feature docs belong under `docs/` and should be linked from `sidebars.ts`.

## Minimal standalone MDX page

```mdx title="src/pages/example.mdx"
---
title: Example Standalone Page
---

# Example

Use this for content that should not appear in the docs sidebar.
```

## Checklist

1. Confirm the page really does not belong in `docs/`.
2. Keep URL naming stable for external links.
3. Build docs and verify there are no broken references.
