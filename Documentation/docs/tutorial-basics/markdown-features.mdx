# Markdown and MDX Standards

Use this style guide for consistent FCTF docs authoring.

## Front Matter

Each page should include metadata:

```yaml
---
title: Page Title
description: One-line summary
sidebar_position: 1
---
```

## Headings

- Use sentence-case headings.
- Keep hierarchy consistent (`#`, `##`, `###`).
- Prefer short, operation-focused section titles.

## Links

- Use relative doc links inside `docs/`.
- Prefer linking to canonical pages instead of duplicate content.

Example:

```md
- [Challenge Operations](../product-and-features/admin/challenge-operations)
```

## Images

- Store screenshots under `static/img/...`.
- Use descriptive alt text.
- Prefer PNG for UI screenshots.

Example:

```md
![Contestant instances page](/img/contestant/InstancesPage.png)
```

## Code Blocks

- Use fenced code blocks with language identifiers.
- Keep examples copy-ready.

```bash
npm run build
```

## Admonitions

Use admonitions for warnings, tips, and constraints:

```md
:::tip[Operational note]
Keep screenshots synchronized with the latest UI release.
:::
```

## MDX usage policy

Use MDX components only when needed for clarity.

Avoid unnecessary interactive widgets in operational docs.
